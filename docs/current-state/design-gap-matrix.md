Acceptance rule:
- `done`: the design requirement is implemented with non-mock behavior and repo-local evidence.
- `open`: repo-local work is still required, or only partial/mock/provider-alias behavior exists.
- `blocked`: the next step cannot proceed in this repo without a named external dependency; the blocker must be stated explicitly.
